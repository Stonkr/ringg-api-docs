## Screenshot Guidelines

- **Resolution**: Use 1920x1080 or higher resolution
- **Format**: PNG format for best quality
- **Content**: Ensure no sensitive data (real phone numbers, credentials) is visible
- **UI State**: Show clean, production-ready UI states
- **Annotations**: Add callouts or highlights for important elements when helpful

## File Naming Convention

Use kebab-case for file names to match the references in the documentation:
- `add-provider-form.png` (not `Add Provider Form.png`)
- `number-marketplace.png` (not `numberMarketplace.png`)

## Image Optimization

Before adding images:
1. Crop to show relevant UI sections
2. Optimize file size while maintaining clarity
3. Ensure images are accessible (good contrast)
4. Test images render correctly in both light and dark themes
