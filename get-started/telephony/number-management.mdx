## Overview

Ringg provides flexible number management options for your telephony integration. You can either import existing phone numbers from your telephony provider or purchase new numbers directly through the Ringg dashboard.

## Number Management Options

  <Card title="Import Numbers" icon="download" href="#importing-numbers">

  </Card>
  <Card title="Purchase Numbers" icon="shopping-cart" href="#purchasing-numbers">
    Buy new phone numbers directly through Ringg's marketplace
  </Card>
</CardGroup>

## Importing Numbers

### Setting Up Your Telephony Provider

Before importing numbers, you need to connect your telephony provider account to Ringg.


### Import Numbers from Provider

Once your provider is connected, you can import your existing numbers:

<Steps>
  <Step title="Go to Numbers Section">
    Navigate to **Numbers** → **Import Numbers** in your Ringg dashboard
  </Step>
   <Step title="Select Provider Account">
    Choose the telephony provider account you want to import numbers from

![Number Dashbaord](/images/number_dashboard.png)
  </Step>

  <Step title="Select Numbers to Import">
    Choose which numbers you want to import (you can select all or specific numbers)
    ![Import Numbers](/images/import_numbers.png)
  </Step>
  <Step title="Import Numbers">
    Click **"Import Selected Numbers"** to complete the import process
    ![Number Dashbaord](/images/number_dashboard.png)
  </Step>
</Steps>


## Purchasing Numbers

Ringg's marketplace allows you to purchase new phone numbers from multiple providers with competitive pricing and instant setup.

### Search and Purchase Numbers

<Steps>
   <Step title="Go to Numbers Section">
    Navigate to **Numbers** → **Import Numbers** in your Ringg dashboard
  </Step>
   <Step title="Select Provider Account">
    Choose the telephony provider account you want to import numbers from

![Number Dashbaord](/images/number_dashboard.png)
  </Step>

  <Step title="Buy Number">
    View available numbers with pricing from different providers
    ![Number Dashbaord](/images/buy_numbers.png)
  </Step>
  <Step title="Use Numbers">
Use the number 
  </Step>
</Steps>



![Exotel Dashboard](/images/number_dashboard.png)



### Attach the number in an inbound agent

#### Release Numbers

When you no longer need a number, you can release it through the dashboard:

<Steps>
  <Step title="Select the telephony ">
    Go to your numbers list and click on the number you want to release
  </Step>
  <Step title="Choose the number">
    Select whether to release from Ringg only or also from the telephony provider
  </Step>
</Steps>

![Release Number Dialog](/images/attach_inbound_number.png)

## Best Practices

<AccordionGroup>
  <Accordion title="Number Management">
    - Regularly audit your numbers and release unused ones
    - Use descriptive names for imported numbers
    - Monitor usage to optimize costs
    - Keep provider credentials secure when importing
  </Accordion>
  
  <Accordion title="Cost Optimization">
    - Compare pricing between providers when purchasing
    - Consider regional numbers for better rates
    - Release numbers during low-usage periods
    - Use Ringg's marketplace for competitive pricing
  </Accordion>
  
  <Accordion title="Security">
    - Validate all number operations
    - Use proper authentication for provider APIs
    - Monitor for unauthorized number changes
    - Implement proper access controls
  </Accordion>
</AccordionGroup>


## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Import failures">
    - Verify provider API credentials are correct
    - Check if numbers are already imported
    - Ensure sufficient permissions in provider account
  </Accordion>
  
  <Accordion title="Purchase failures">
    - Verify sufficient credits in Ringg account
    - Check if number is still available
    - Ensure proper country/region selection
  </Accordion>
  
  <Accordion title="Configuration issues">
    - Verify assistant ID exists
    - Check webhook URL accessibility
    - Ensure proper number status (active)
  </Accordion>
</AccordionGroup>
